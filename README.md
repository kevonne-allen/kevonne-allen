Networking and Content Delivery
Amazon API Gateway
Managed service to create, publish, and manage APIs. DevOps use: Frontend for Lambda functions, expose microservices, throttle requests. Exam critical: REST vs. HTTP vs. WebSocket APIs, throttling, caching, integration with Lambda/Step Functions, stage variables, deployment stages, canary deployments. Exam tip: Use for serverless API deployments (Domain 1.4).

AWS Client VPN
Managed VPN service for remote access. DevOps use: Secure remote access to AWS resources. Exam tip: Low priority; know it provides VPN access for users.

Amazon CloudFront
Global content delivery network (CDN). DevOps use: Cache static/dynamic content at edge locations, reduce latency, DDoS protection. Exam critical: Origin configurations (S3, ALB), cache behaviors, signed URLs/cookies (restrict access), Lambda@Edge (run code at edge). Exam tip: Use for global application resilience (Domain 3.1).

Elastic Load Balancing (ELB)
Distributes traffic across targets (EC2, containers, Lambda). DevOps use: High availability, health checks, SSL termination. Exam critical: ALB (HTTP/HTTPS, path-based routing, target groups), NLB (TCP/UDP, static IP), health checks, connection draining, sticky sessions, cross-zone load balancing. Must know: ALB integration with CodeDeploy for blue/green deployments.

AWS PrivateLink
Private connectivity between VPCs and AWS services without internet exposure. DevOps use: Secure service-to-service communication, VPC endpoints. Exam tip: Understand for secure multi-account architectures; avoid internet gateways.

Amazon Route 53
Scalable DNS and domain registration service. DevOps use: DNS routing, health checks, failover routing. Exam critical: Routing policies (simple, weighted, latency, failover, geolocation, geoproximity, multi-value), health checks (trigger failover), alias records (AWS resource integration). Must know: How to implement multi-region failover using health checks (Domain 3.3).

AWS Site-to-Site VPN
Encrypted connection between on-premises and AWS. DevOps use: Hybrid cloud connectivity. Exam tip: Understand for hybrid deployment scenarios; Transit Gateway integration for multi-VPC connectivity.

AWS Transit Gateway
Connects VPCs and on-premises networks through central hub. DevOps use: Simplify complex network topologies, centralize routing. Exam tip: Understand for multi-account/multi-VPC architectures (Domain 2.2).

Amazon VPC
Isolated virtual network in AWS. DevOps use: Network isolation, control IP addressing, subnets, routing, security. Exam critical: Subnets (public/private), route tables, internet gateways, NAT gateways, security groups (stateful), NACLs (stateless), VPC peering, VPC endpoints (Gateway/Interface). Must know: Security group vs. NACL differences, VPC Flow Logs for network monitoring.

Security, Identity, and Compliance
AWS Certificate Manager (ACM)
Provision, manage, and deploy SSL/TLS certificates. DevOps use: Enable HTTPS on ELB, CloudFront, API Gateway. Exam critical: Automatic renewal, integration with AWS services, certificate validation methods (DNS/email). Exam tip: Use for encrypting data in transit.

AWS CloudHSM
Hardware security module for cryptographic key management. DevOps use: Store encryption keys with dedicated hardware control. Exam tip: Understand vs. KMS (CloudHSM gives you exclusive hardware control; use for regulatory compliance requiring dedicated HSM).

Amazon Cognito
User authentication and authorization for applications. DevOps use: Add sign-up/sign-in to apps, federate identities. Exam tip: Low priority for DevOps exam; know it handles user authentication.

Amazon Detective
Analyze and investigate security findings. DevOps use: Root cause analysis of security incidents using machine learning. Exam tip: Understand it investigates GuardDuty/Security Hub findings (Domain 6.3).

AWS Directory Service
Managed Microsoft Active Directory. DevOps use: Integrate AWS resources with on-premises AD, enable SSO. Exam tip: Understand for hybrid environments requiring AD integration.

Amazon GuardDuty
Threat detection service using machine learning. DevOps use: Detect unauthorized access, compromised instances, malicious IPs. Exam critical: Continuous monitoring of CloudTrail, VPC Flow Logs, DNS logs; findings sent to EventBridge for automated response. Must know: Integration with EventBridge/Lambda for automated remediation (Domain 6.3).

AWS IAM Identity Center
Centralized access management for multiple AWS accounts (formerly AWS SSO). DevOps use: Single sign-on across accounts, integrate with external identity providers. Exam critical: Permission sets, multi-account access, SAML/OIDC federation. Exam tip: Use for centralized identity management in multi-account environments (Domain 6.1).

AWS Identity and Access Management (IAM)
Manage access to AWS resources. DevOps use: Control who can do what on which resources. Exam critical: Users, groups, roles, policies (identity-based, resource-based), policy evaluation logic, least privilege principle, IAM roles for cross-account access, instance profiles (EC2 role assumption), STS for temporary credentials, MFA, permissions boundaries, service control policies. Must know: When to use roles vs. users, policy troubleshooting, cross-account role assumption patterns.

Amazon Inspector
Automated security assessment for EC2 and ECR. DevOps use: Scan for vulnerabilities, configuration issues, compliance violations. Exam critical: Automated scanning of EC2 instances and container images, findings sent to Security Hub/EventBridge. Exam tip: Use for continuous security assessment (Domain 6.3).

AWS Key Management Service (AWS KMS)
Managed encryption key service. DevOps use: Encrypt data at rest and in transit, integrate with S3, EBS, RDS, Secrets Manager. Exam critical: Customer managed keys (CMK) vs. AWS managed keys, key policies, automatic key rotation, envelope encryption, grant mechanism for temporary access. Must know: How KMS integrates with Secrets Manager for encryption (Domain 6.2).

Amazon Macie
Discover and protect sensitive data (PII) using machine learning. DevOps use: Automated data classification, detect exposed S3 data. Exam tip: Understand for data protection automation (Domain 6.2); scans S3 for sensitive data.

AWS Network Firewall
Managed firewall service for VPC traffic filtering. DevOps use: Implement stateful firewall rules, intrusion detection/prevention. Exam tip: Understand for network-level security (Domain 6.2); choose when advanced packet inspection needed.

AWS Resource Access Manager (AWS RAM)
Share AWS resources across accounts. DevOps use: Share VPC subnets, Transit Gateways, Route 53 Resolver rules across accounts. Exam tip: Use for multi-account resource sharing (Domain 2.2).

AWS Secrets Manager
Store and rotate database credentials, API keys, secrets. DevOps use: Automate secret rotation, eliminate hardcoded credentials. Exam critical: Automatic rotation for RDS/Aurora/Redshift, integration with Lambda for custom rotation, encryption with KMS, integration with CodeBuild/CodePipeline for build secrets. Must know: Difference vs. Parameter Store (Secrets Manager has rotation, costs more; Parameter Store is free tier, no native rotation).

AWS Security Hub
Centralized security and compliance view across accounts. DevOps use: Aggregate findings from GuardDuty, Inspector, Macie, Config; automated compliance checks. Exam critical: Security standards (CIS, PCI-DSS), automated remediation using EventBridge, multi-account aggregation. Exam tip: Use for centralized security monitoring (Domain 6.3).

AWS Security Token Service (AWS STS)
Create temporary security credentials. DevOps use: Enable cross-account access, federated user access, EC2 instance role credentials. Exam critical: AssumeRole (cross-account), GetSessionToken, temporary credentials for least privilege. Exam tip: Understand role assumption patterns for CI/CD pipelines (Domain 1.1).

AWS Shield
DDoS protection (Standard free, Advanced paid). DevOps use: Protect applications from DDoS attacks. Exam tip: Shield Standard is automatic; Shield Advanced provides enhanced protection and DDoS cost protection. Use with CloudFront/Route 53.

AWS WAF
Web application firewall to filter HTTP/HTTPS traffic. DevOps use: Protect APIs and web apps from SQL injection, XSS, bot traffic. Exam critical: Web ACLs, rules (rate-based, IP sets, managed rule groups), integration with ALB, API Gateway, CloudFront. Exam tip: Use for application-layer security (Domain 6.2).

Serverless
AWS Lambda
Run code without provisioning servers (event-driven functions). DevOps use: Process events, automate tasks, API backends, data transformation. Exam critical: Triggers (S3, EventBridge, ALB, API Gateway, SQS), execution roles, environment variables, layers (shared code), concurrency limits, timeout/memory configuration, VPC integration, Lambda versions/aliases for blue/green deployments. Must know: How to deploy Lambda with CodeDeploy for canary/linear deployments (Domain 1.4).

AWS Serverless Application Model (AWS SAM)
Framework for building serverless applications (extends CloudFormation). DevOps use: Define Lambda, API Gateway, DynamoDB in simplified syntax, local testing. Exam critical: SAM templates (Transform: AWS::Serverless-2016-10-31), SAM CLI for local testing, deployment via CodePipeline. Exam tip: Choose SAM for simplified serverless IaC (Domain 2.1).

Amazon Simple Notification Service (Amazon SNS)
Pub/sub messaging service. DevOps use: Send notifications (email, SMS, HTTP), fan-out messages to multiple subscribers. Exam critical: Topics, subscriptions (email, SQS, Lambda, HTTP), message filtering, integration with CloudWatch alarms, EventBridge. Must know: SNS ‚Üí SQS fan-out pattern for decoupled architectures.

Amazon Simple Queue Service (Amazon SQS)
Managed message queue service. DevOps use: Decouple microservices, buffer requests, asynchronous processing. Exam critical: Standard (best-effort ordering, at-least-once delivery) vs. FIFO (exactly-once, ordered), visibility timeout, dead-letter queues (DLQ), long polling, integration with Lambda (event source). Must know: When to use SQS vs. SNS (SQS for work queues, SNS for notifications).

AWS Step Functions
Orchestrate workflows using state machines. DevOps use: Coordinate multiple Lambda functions, long-running workflows, error handling, retries. Exam critical: State types (Task, Choice, Parallel, Wait, Fail, Succeed), integration with Lambda, ECS, DynamoDB, SNS, standard vs. express workflows. Exam tip: Use for complex automation workflows (Domain 2.3).

Storage
AWS Backup
Centralized backup service across AWS services. DevOps use: Automate backups for EBS, RDS, DynamoDB, EFS, S3, cross-region backup. Exam critical: Backup plans, backup vaults, cross-region/cross-account backups, point-in-time recovery. Must know: Use for automated disaster recovery (Domain 3.3).

Amazon Elastic Block Store (Amazon EBS)
Block storage for EC2 instances. DevOps use: Persistent storage for EC2, snapshots for backups. Exam critical: Volume types (gp3, io2, st1, sc1), snapshots (incremental backups to S3), encryption, volume resizing, lifecycle policies for snapshot management. Exam tip: Understand snapshot automation with Data Lifecycle Manager.

AWS Elastic Disaster Recovery
Continuous replication for fast disaster recovery (formerly CloudEndure). DevOps use: Minimize downtime and data loss with continuous replication to AWS. Exam tip: Understand for DR scenarios requiring low RTO/RPO.

Amazon Elastic File System (Amazon EFS)
Managed NFS file system for Linux workloads. DevOps use: Shared storage for EC2, ECS, Lambda, scales automatically. Exam critical: Lifecycle policies (move files to Infrequent Access), mounting across AZs, performance modes (General Purpose vs. Max I/O). Exam tip: Use when multiple compute resources need shared file access.

Amazon FSx for Lustre
High-performance file system for compute-intensive workloads. DevOps use: HPC, machine learning, media processing. Exam tip: Low priority; know it's for high-performance computing.

Amazon FSx for NetApp ONTAP
Fully managed NetApp ONTAP file system. DevOps use: Migrate NetApp workloads to AWS. Exam tip: Low priority unless NetApp-specific scenario.

Amazon FSx for OpenZFS
Fully managed OpenZFS file system. DevOps use: Migrate ZFS workloads to AWS. Exam tip: Low priority; niche use cases.

Amazon FSx for Windows File Server
Fully managed Windows file system (SMB protocol). DevOps use: Shared storage for Windows workloads, Active Directory integration. Exam tip: Use when scenario requires Windows file shares.

Amazon S3
Object storage service (scalable, durable, highly available). DevOps use: Store artifacts, logs, backups, static websites, CloudFormation templates. Exam critical: Bucket policies, versioning, lifecycle policies (transition to Glacier, expiration), S3 Event Notifications (trigger Lambda), replication (cross-region, same-region), encryption (SSE-S3, SSE-KMS, SSE-C), presigned URLs, access points, S3 Object Lock (compliance/governance mode). Must know: How to use S3 for artifact storage in CodePipeline (Domain 1.3).

Amazon S3 Glacier
Low-cost archival storage. DevOps use: Long-term backup retention, compliance archives. Exam tip: Understand lifecycle policies to transition S3 objects to Glacier for cost optimization.

AWS Storage Gateway
Hybrid storage integration between on-premises and AWS. DevOps use: Backup on-premises data to AWS, cached access to S3. Exam tip: Understand for hybrid cloud scenarios; File Gateway (S3), Volume Gateway (EBS snapshots), Tape Gateway (VTL).
